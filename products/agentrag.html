Agent Library

Each agent is defined by a single .md file with YAML frontmatter. Register via POST /api/skills/register — credentials resolved securely from DB, never in plain config.

Open-source alternative to OpenClaw & Claude Computer Use

AgentRAG.io brings the same skill-based agent paradigm — self-hosted, fully controllable, multi-provider. Drop a .md file to create an agent. Connect SSH or WinRM hosts once, run anywhere. No vendor lock-in, no usage caps.

Skill .md file examples

One file = one agent. The YAML frontmatter configures routing, MCP servers and memory. The Markdown body is the system prompt — including the ## Mapping section that defines the commands the agent understands at execution. 

---
skill_id: ssh_admin
name:     ssh_admin
type:     agent
mcp_servers: [ssh]
description: "Administration Linux distante via SSH"
version: "1.0"
tags: [linux, ssh, admin, infra]
slash_command: /ssh
at_command: true
host_required: true
memory_scope: session+global
---

## Objective
You are an expert Linux administrator. Translate user requests
into precise SSH commands and report results clearly.

## Mapping
- restart <service>  → systemctl restart <service>
- stop <service>     → systemctl stop <service>
- check <service>    → systemctl status <service> --no-pager
- logs <service>     → journalctl -u <service> -n 50 --no-pager
- disk               → df -h
- memory             → free -h
- processes          → ps aux --sort=-%mem | head -20
- ports              → ss -tlnp
- cleanup            → apt autoremove -y && journalctl --vacuum-time=7d

## First run on unknown host
Execute get_system_info to detect OS and store globally.

## Response format
✅ Success | ⚠️ Warning | ❌ Error + stderr + suggested fix

---
skill_id: postgres_ops
name:     postgres_ops
type:     agent
mcp_servers: [ssh]
description: "PostgreSQL operations via psql over SSH"
version: "1.0"
tags: [postgres, postgresql, db, sql, ssh]
host_required: true
memory_scope: session+global
---

## Objective
You are a PostgreSQL DBA. Diagnose and operate PostgreSQL
instances via psql commands executed over SSH.

## Mapping
- status       → systemctl status postgresql --no-pager
- restart      → systemctl restart postgresql
- connections  → psql -U postgres -c "SELECT count(*), state \
                   FROM pg_stat_activity GROUP BY state"
- db sizes     → psql -U postgres -c "SELECT datname, \
                   pg_size_pretty(pg_database_size(datname)) \
                   FROM pg_database ORDER BY 2 DESC"
- locks        → psql -U postgres -c "SELECT pid, usename, \
                   state, wait_event_type, query \
                   FROM pg_stat_activity \
                   WHERE wait_event_type='Lock'"
- slow queries → psql -U postgres -c "SELECT pid, \
                   now()-query_start AS duration, query \
                   FROM pg_stat_activity WHERE state='active' \
                   ORDER BY duration DESC LIMIT 10"
- vacuum       → psql -U postgres -c "VACUUM ANALYZE"
- logs         → tail -n 100 /var/log/postgresql/postgresql-*.log
- disk         → df -h /var/lib/postgresql

## Response format
✅ Success | ⚠️ Warning | ❌ Error + psql stderr + fix

skill_id: gitea_code_generator
type:     skill
mcp_servers: ["gitea", "linter"]
description: "Generates code, lints, commits and opens PRs on
  a self-hosted Gitea repository"
version: "1.0"
tags: [code, git, gitea, codegen, ci, automation]
project_required: false
---

## Objective
You are an expert code-generation agent. You work on a repository
hosted on a self-hosted Gitea instance (GitHub v3-compatible API).

## Mandatory workflow
clone_repository → get_repo_tree → [get_file_content]
→ LLM generation → lint/format
→ commit_and_push → [create_pull_request if auto_create_pr=true]

## Generation rules
- Respect existing code style (indent, naming, patterns)
- Never overwrite an entire file to change a single function
- Docstring required on all new public functions
- Always work on target_branch, never directly on main

## Branching
- base_branch — source branch (default: main)
- target_branch — AI feature branch (default: ai-feature)
- Create target branch via create_branch if it does not exist

## Response format
{
  "files_modified": ["path/to/file.py"],
  "commit_hash": "abc1234",
  "branch": "ai-feature",
  "pr_url": null,
  "summary": "Description of changes"
}

MCP Servers

AgentRAG connects agents to external services via Model Context Protocol servers. Declare the MCP server in your .md frontmatter — the backend resolves credentials and routes tool calls automatically.

SSH Server ssh
stdio

Execute commands on any Linux host over SSH. Supports key-based auth, passphrase keys, and per-session memory. Hosts registered once in DB — never in plain config.

exec_command get_system_info upload_file download_file list_directory
WinRM Server winrm
stdio

Execute PowerShell commands on Windows hosts via WinRM. Supports NTLM, Kerberos, and Basic auth. Service management, Event Log queries, process control.

exec_powershell get_system_info restart_service list_services get_event_logs get_running_processes
Gitea Server gitea New
HTTP

Full repository operations on a self-hosted Gitea instance (GitHub v3 API). Clone, read files, update content, create branches and pull requests — all without leaving AgentRAG.

clone_repository get_repo_tree get_file_content update_file commit_and_push create_branch create_pull_request
GitHub Server github
HTTP

Interact with GitHub repositories — code review automation, PR creation, issue management, branch operations and content updates via the GitHub REST API.

get_file_content update_file create_pull_request list_issues create_review
DataGouv datagouv New
HTTP

Search and explore the French open data catalogue — data.gouv.fr REST API v1 & v2. No authentication required. Search datasets, fetch resources, browse DINUM topics and organisations.

search_datasets get_dataset search_organizations list_topics get_topic
Linter Server linter New
stdio

Run linters and formatters on generated code before committing. Supports Python (ruff, black, flake8), JavaScript (eslint, prettier), and shell scripts. Used automatically by the Gitea code generator.

lint_file format_file lint_directory check_syntax
Test Runner test_runner
stdio

Execute test suites as part of agent workflows. Supports pytest, jest, and custom commands. Results are parsed and returned as structured JSON — failures block commit when enabled.

run_tests run_single_test get_coverage
Transport: stdio — local subprocess HTTP — remote REST API SSE — server-sent events
